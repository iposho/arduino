# Arduino / ESP32 Projects

Мои домашние проекты на ESP32 для мониторинга климата и качества воздуха.

## Проекты

### 1. esp32_balcony_pms5003_bme280 — Балконный датчик

ESP32, который измеряет:

| Датчик | Параметры |
|--------|-----------|
| **PMS5003** | PM1.0, PM2.5, PM10 (со статистикой: min, max, median, mean, stddev) |
| **BME280** | Температура, влажность, давление |

- Отправляет данные в **Supabase** каждые 5 минут
- Шлёт оповещения в **Telegram** при выходе за границы
- PMS5003 просыпается, делает замеры, засыпает — экономия ресурса лазера

**Пины:**
- PMS5003 RX → GPIO16, TX → GPIO17
- BME280 — I²C (адрес 0x76)

### 2. esp32_flat_bme280 — Дисплей на кухне/в комнате

ESP32 с **1.8" TFT-дисплеем (ST7735)** и **BME280**.

- Отображает температуру, влажность, давление (локальные и с балкона)
- AQI-индекс на основе данных PMS5003 с балкона
- Несколько экранов: Home, Outdoor, AQI, время/дата
- Управление: джойстик (вверх/вниз/влево/вправо/кнопка)
- Фильтрация показаний BME280 (EMA-фильтр)
- Автоматическое переключение на экран времени и back

**Пины TFT:**
| TFT | GPIO |
|-----|------|
| CS  | 5   |
| DC  | 17  |
| RST | 2   |
| SCLK | 18 |
| MOSI | 23 |

**BME280** — I²C (GPIO21-SDA, GPIO22-SCL)

## Настройка

1. Скопируйте `secrets.example.h` → `secrets.h` в папке нужного проекта
2. Заполните Wi-Fi, Supabase, Telegram данные
3. Откройте `.ino` в Arduino IDE или PlatformIO
4. Установите зависимости:
   - `Adafruit BME280 Library`
   - `Adafruit ST7735` / `Adafruit ST7789` (для проекта с дисплеем)
   - `ESP8266 HTTPClient` / `ArduinoHttpClient`
   - `ArduinoJson`

## Supabase

Используется таблица `weather_logs` со схемой:

| Колонка | Тип | Описание |
|---------|-----|----------|
| temp | float8 | Температура (°C) |
| humidity | float8 | Влажность (%) |
| pressure | float8 | Давление (гПа) |
| pm1 | float8 | PM1.0 (мкг/м³) |
| pm25 | float8 | PM2.5 (мкг/м³) |
| pm10 | float8 | PM10 (мкг/м³) |
| location | text | `balcony` или `flat` |
